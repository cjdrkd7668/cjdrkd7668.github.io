About Me

안녕하세요, 제 포트폴리오 페이지를 방문해주셔서 감사합니다.
저는 대학교에서 광고와 경영학을 전공하다, 직접 서비스를 만드는 개발자가 되고 싶어 웹 서비스 개발자의 진로를 걷고있습니다.

어려서부터 사색이 참 많았는데, 요즘은 어떻게 해야 좋은 개발자가 될 수 있을까 생각하곤 합니다. 처음 코딩을 배울 때는 의도한 기능을 정확하게 구현하는 것이 개발자의 덕목이구나 생각했었는데, 네 번의 프로젝트를 진행하면서는 성공적으로 프로젝트를 완수하기 위해 팀과 소통할 수 있는, 호출이나 선언의 위치를 팀원이 예상할 수 있는 코드를 작성할 줄 아는 사람이 좋은 개발자라고 생각했습니다.

그러다 가장 최근에 진행한 프로젝트에서 스프링 프레임워크를 사용하면서 또 한번 생각이 바뀌었습니다. 스프링의 DI, IOC, AOP 개념을 지키기 위해 코드를 작성하니 자연스레 읽기 좋은 코드를 작성할 수 있었더랬죠. 웹 개발자를 지망하는 저로서, 전자정부 표준프레임워크와 같은 프레임워크를 업무에 적용하게 된다면 제가 먼저 해야할 공부는 해당 프레임워크에 대한 것이겠지만, 그것만으로는 좋은 개발자가 되기엔 부족할 것이라 생각합니다.

그래서 현재는 좋은 개발자에 대해 처음 떠올렸던 '의도한 기능을 정확히 구현하는 사람'이라는 대답에 가장 공감하고 있습니다. 고객의 요구, 의도를 실현하기 위해 데이터 간의 관계를 어떻게 구성할것인지, 어떤 모듈을 구현할지, 어떤 로직이 필요할지.. 고민하고 떠올려내는 능력이야말로 좋은 개발자가 갖추어야 할 능력이라고 생각합니다.

고민하는 개발자, 흔한 타이틀이지만 항상 좋은 개발자이자 팀원이 되기위해 노력하고 있습니다. 잘 부탁드립니다!

"항상 개선하기 위해 고민하는, 든든한 팀원이 되겠습니다."

프로젝트 명 : 쌍용리빙 프로젝트 개발 기간 : 2021.03.04 ~ 2021.03.22 개발 인원 : 4명 (팀장) 개요 :
Spring Framework(이하 스프링)를 처음 사용해본 프로젝트입니다. 이 프로젝트가 가장 최근에 참여한 프로젝트입니다. 이전까지는 팀원 간의 코드 작성 스타일(추상화 정도 등)이 달라서 서로의 코드를 알아보기가 어려웠지만, 스프링을 사용하면서 이 점이 훨씬 개선할 수 있었습니다. 덤으로 매퍼 인터페이스로 데이터 연결 작업이 간단해진 것도 유용했습니다. 깃허브 원격 저장소
프로젝트 명 : 교육개발센터 CRUD 개발 기간 : 2020.12.02 ~ 2020.12.24 개발 인원 : 6명 개요 :
오라클 데이터베이스와 JDK의 JDBC 인터페이스를 처음 사용해본 프로젝트입니다. 또한 처음으로 MVC 디자인 패턴을 프로젝트에 적용해보기도 하였습니다. 이 프로젝트 덕분에 머릿속에 클래스를 왜 사용하는 지 정리가 되었습니다. 그리고 이 때부터 저장 프로시저를 작성하는 것에 흥미가 생겨서 앞서 소개한 두 웹 프로젝트를 진행할 때 도움이 많이 되었습니다.
